## 2. 릴리스 APK 빌드 (배포용)

### 2-1. 서명 키 생성 (한 번만)

```powershell
cd C:\Users\yejin\Desktop\sesac_final\UsersyejinDesktopSeSAC_final\inspection_app\android
keytool -genkey -v -keystore app\upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload -storepass <비밀번호> -keypass <비밀번호> -dname "CN=Inspector, OU=Shipyard, O=Sesac, L=Seoul, ST=Seoul, C=KR"
```

`<비밀번호>` 부분을 원하는 비밀번호로 교체하세요 (최소 6자). 비밀번호는 **꼭 기억**하세요. 잃어버리면 같은 앱을 업데이트할 수 없습니다.

> ⚠️ **PowerShell 주의**: `keytool`을 옵션 없이 실행하면 비밀번호 대화형 입력 시 `java.lang.NullPointerException`이 발생합니다. 위처럼 `-storepass`·`-keypass`·`-dname`을 한 줄에 명시하세요.

> ⚠️ `upload-keystore.jks`는 **절대 git에 올리지 마세요.** `.gitignore`에 추가.

`android\.gitignore`에 다음 추가:
```
key.properties
**/*.jks
```

> **참고**: Flutter 기본 생성 `.gitignore`에 `**/*.jks`와 `key.properties`가 이미 포함되어 있으면 별도 추가 불필요. `**/*.jks`는 `**/upload-keystore.jks`를 포함하는 더 넓은 패턴.

### 2-2. 키 설정 파일

`android\key.properties`:
```
storePassword=위에서설정한비번
keyPassword=위에서설정한비번
keyAlias=upload
storeFile=upload-keystore.jks
```

### 2-3. Gradle 설정

`android\app\build.gradle.kts` 안의 `android { ... }` 블록 위에 추가:

```kotlin
import java.util.Properties
import java.io.FileInputStream

val keystoreProperties = Properties()
val keystorePropertiesFile = rootProject.file("key.properties")
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(FileInputStream(keystorePropertiesFile))
}
```

`android { ... }` 안의 `defaultConfig { ... }` 아래에 추가:

```kotlin
signingConfigs {
    create("release") {
        keyAlias = keystoreProperties["keyAlias"] as String
        keyPassword = keystoreProperties["keyPassword"] as String
        storeFile = file(keystoreProperties["storeFile"] as String)
        storePassword = keystoreProperties["storePassword"] as String
    }
}

buildTypes {
    release {
        signingConfig = signingConfigs.getByName("release")
        isMinifyEnabled = false
        isShrinkResources = false
    }
}
```

### 2-4. 릴리스 빌드

```powershell
flutter build apk --release --dart-define=API_BASE_URL=http://<PC-LAN-IP>:8000/api
```

> ⚠️ **실기기 배포 시 주의**: `--dart-define=API_BASE_URL`을 명시하지 않으면 앱이 에뮬레이터용 기본값(`10.0.2.2`)으로 빌드되어 실기기에서 서버에 연결할 수 없습니다. `<PC-LAN-IP>`는 `ipconfig`로 확인한 PC의 LAN IP로 교체하세요 (예: `172.16.210.34`).

결과물: `build\app\outputs\flutter-apk\app-release.apk` (약 40MB — 디버그보다 작음)

### 2-5. ABI 분리 (선택, 더 작은 APK)

```powershell
flutter build apk --release --split-per-abi
```

결과:
- `app-arm64-v8a-release.apk` (~25MB) — 대부분 폰
- `app-armeabi-v7a-release.apk` — 구형 폰
- `app-x86_64-release.apk` — 에뮬레이터/일부

운영에서는 `arm64-v8a`만 배포해도 90% 이상 커버.

## 3. 사내 배포 (간단 가이드)

### 3-1. APK 공유 방법 (MDM 도입 전)

1. **사내 파일 서버 / Google Drive / OneDrive**에 APK 업로드
2. 단축 URL 생성 (`bit.ly` 등)
3. 검사원이 폰 브라우저로 URL 접속 → APK 다운로드 → 설치
4. 설치 시 다음 경고가 뜨면 허용:
   - "출처를 알 수 없는 앱" → 설정 → 해당 브라우저/파일관리자 → "이 출처 허용"

### 3-2. MDM 도입 시 (4주차 작업)

MDM(Mobile Device Management) — 사내 IT가 폰을 중앙 관리. APK를 한 번 등록하면 모든 검사원 폰에 자동 푸시 설치.

대표 제품:
- VMware Workspace ONE
- Microsoft Intune
- Samsung Knox Manage (삼성 폰 위주)

**[19_4주차_MDM배포_JWT안정화.md](19_4주차_MDM배포_JWT안정화.md)** 에서 본격 다룸. 사내 IT팀과 협업하여 4주차에 도입.

### 3-3. Play Console 내부 테스트 (대안)

MDM이 아직 없고 사내망 APK 배포가 막혀있다면:
1. Google Play Console에 무료 계정 (회사 계정 권장)
2. 내부 테스트 트랙에 APK 업로드
3. 검사원 Gmail을 테스터로 등록
4. Play Store 앱에서 베타 테스터로 참여 → 자동 업데이트

## 5. 트러블슈팅 종합 카탈로그

자주 만나는 오류와 해결책을 한 자리에 모았습니다. 막혔을 때 Ctrl+F로 검색하세요.

### Flutter 빌드·실행

#### `flutter doctor` 빨간 X
| 메시지 | 해결 |
|---|---|
| `Android toolchain - develop for Android devices: ✗` | Android Studio 설치 + `flutter doctor --android-licenses` |
| `cmdline-tools component is missing` | Android Studio → SDK Manager → SDK Tools → "Android SDK Command-line Tools" 설치 |
| `Unable to locate Android SDK` | `flutter config --android-sdk "C:\Users\<user>\AppData\Local\Android\Sdk"` |
| `Java binary at: ...openjdk\bin\java` 와 Gradle 호환 안 됨 | Android Studio가 설치한 JBR 사용. `flutter config --jdk-dir "C:\Program Files\Android\Android Studio\jbr"` |

#### Gradle 빌드 무한 대기 / 실패
- 첫 빌드는 5~10분이 정상. 더 걸리면 `%USERPROFILE%\.gradle\` 손상 의심
- `Remove-Item -Recurse -Force $env:USERPROFILE\.gradle` 후 재빌드
- 회사 방화벽이 Gradle 다운로드 막을 수 있음 → VPN 사용 또는 IT 문의

#### `build_runner` 충돌
```powershell
dart run build_runner build --delete-conflicting-outputs
```
- 그래도 안 되면 `.dart_tool/` 폴더 삭제 후 `flutter pub get`

#### `MissingPluginException` 런타임 에러
- 네이티브 플러그인 추가 후 핫 리로드만 했음. **전체 재시작 필요** (Stop → Run)

#### Hot Reload가 일부 변경을 못 잡음
- `main.dart`나 위젯 트리 최상단 변경은 핫 리스타트(`Shift+R`)가 필요
- 그래도 이상하면 전체 재시작

### 네트워크·백엔드

#### "Connection refused" (에뮬레이터)
- 백엔드가 안 떠 있음 — `uvicorn` 창 확인
- baseUrl이 `localhost`가 아닌 `10.0.2.2`인지 확인 (에뮬레이터에서 호스트 PC 가리키는 특수 IP)

#### "Connection refused" (실기기)
- 같은 Wi-Fi에 있는지 확인
- baseUrl이 PC의 LAN IP 인지 (`ipconfig`로 확인, `192.168.x.x` 형태)
- Windows Defender 방화벽이 8000번 막고 있는지 확인 → 인바운드 허용
- 백엔드를 `uvicorn ... --host 0.0.0.0` 로 띄웠는지 (`127.0.0.1`로 띄우면 외부 접근 불가)

#### "Cleartext HTTP traffic not permitted"
- 03번 단계 `AndroidManifest.xml`의 `android:usesCleartextTraffic="true"` 확인
- 운영 시에는 HTTPS로 바꾸고 이 줄 제거

#### JWT 401 무한 루프
- `AuthInterceptor`의 `refreshDio`가 인터셉터를 갖고 있으면 재귀 → `refreshDio`는 깨끗한 새 Dio여야 함
- `Dio.clone()`은 인터셉터까지 복제됨 → 새 `Dio(BaseOptions(...))` 사용

#### "DAILY_SESSION_EXISTS" 영구적 발생
- 진행중 세션을 종료 안 함. DB로 직접:
  ```sql
  UPDATE 검사_세션 SET 세션_상태='완료', 종료_일시=NOW() WHERE 세션_상태='진행중';
  ```

### Docker·DB

#### `docker compose up` 실패 — "port 5432 in use"
```powershell
# 현재 5432번 점유 프로세스 확인
netstat -ano | findstr :5432
# 해당 PID 종료 또는 docker-compose.yml의 포트 매핑 변경
```

#### PostgreSQL이 healthy 상태가 안 됨
```powershell
docker compose logs postgres
```
- `permission denied` → Docker Desktop 재시작
- `initdb: directory not empty` → `docker compose down -v` 후 다시

#### 시드 SQL이 적용되지 않음
- `postgres_data` 볼륨이 이미 존재 (이전 실행 흔적)
- 완전히 처음부터: `docker compose down -v` (`-v`가 핵심)

#### 한글 컬럼명 SQL에서 인용
- SQLAlchemy raw SQL에서는 그냥 쓸 수 있음 (`SELECT 검사원_ID FROM 검사원`)
- 충돌하는 한글 컬럼명 (예: `Grad-CAM_경로`처럼 하이픈 포함)은 큰따옴표로 감싸기: `"Grad-CAM_경로"`

### Python·LLM·AI

#### `torch` 설치 실패 또는 GPU 못 찾음
- CPU만 쓸 거면 `torch==2.4.0+cpu` 명시
- GPU 쓰려면 CUDA 버전 확인 후 PyTorch 공식 사이트의 install 명령 사용

#### BGE-M3 다운로드가 매우 느림
- HuggingFace에서 미리 다운로드 후 mount:
  ```python
  SentenceTransformer("/path/to/local/bge-m3")
  ```

#### Anthropic API "rate limit"
- 무료 티어는 분당 50회 정도. 동시에 너무 많이 호출하지 않게
- 운영에서는 큐 + 재시도 (exponential backoff)

#### pgvector 검색이 비어 있음
- 벡터가 NULL인지 확인: `SELECT 매뉴얼_ID, 내용_벡터 IS NULL FROM 매뉴얼;`
- 시드 스크립트 다시 실행 (`python scripts/seed_manuals.py`)

### 팀원 코드 병합

#### `pubspec.yaml` 패키지 중복 선언
팀원이 올린 코드를 pull했을 때 같은 패키지가 두 번 선언되면 빌드 오류 발생:
```
Because of duplicate key tflite_flutter...
```
`pubspec.yaml`에서 중복된 줄을 찾아 하나만 남긴다.

#### 메서드 중복 선언 (Dart)
같은 파일에 동일한 메서드가 두 번 선언되면 빌드 오류 발생:
```
Error: 'uploadAndInspectWithDevice' is already declared in this scope.
```
중복된 메서드 블록을 찾아 구버전(팀원 추가분)을 삭제하고 최신 버전(sector·subsector·타임아웃 포함)만 남긴다.

### Flutter Riverpod·State

#### State가 갱신 안 됨
- `state = state.copyWith(...)` 형태로 새 객체 할당해야 함 (mutation 금지)
- `freezed` 클래스는 자동 copyWith 제공

#### `ref.read` vs `ref.watch`
- `ref.watch(...)`: 변화 감지 + 위젯 리빌드 (`build` 안에서 사용)
- `ref.read(...)`: 일회성 (콜백 안에서 사용)
- 둘 바꿔 쓰면 무한 리빌드 또는 갱신 누락

#### `dart run build_runner` 충돌
```powershell
dart run build_runner build --delete-conflicting-outputs
```

## 6. 마지막 점검 — 인수 테스트 시나리오

릴리스 APK가 진짜 동작하는지 12단계 골든 패스로 검증:

| # | 단계 | 기대 결과 |
|---|---|---|
| 1 | 앱 첫 설치 후 실행 | 로그인 화면 |
| 2 | 잘못된 비번 입력 | "ID 또는 비밀번호가 올바르지 않습니다." 토스트 |
| 3 | `1` / `김검사` / `test1234` | 탱크 위치 선택 화면 |
| 4 | 탱크 B → 외부 → 지지대 → 확인 | 대시보드, 통계 카드 갱신 |
| 5 | 검사 시작 → 양품 사진 5장 촬영 | 카메라 화면 유지, 카운터 +5. uvicorn에 `/local-result` 5회 로그 |
| 6 | 비행기 모드 ON → 불량 사진 2장 | 토스트 "오프라인 — ...". sqflite에 2건 큐 |
| 7 | 비행기 모드 OFF | 자동 `/offline-batch` 호출, 큐 비워짐 |
| 8 | 검사 이력 → 빨강 카드 | Bottom Sheet, Grad-CAM 토글, Top-3, RAG 가이드 |
| 9 | 결과 처리 → 입력 → 저장 | 토스트 + 검사 이력 복귀, 해당 카드 "완료" |
| 10 | 모든 카드 완료 후 "세션 완료" | 활성화 → 클릭 → 대시보드, 최근 이력 1건 추가 |
| 11 | 설정 → 푸시 OFF → 언어 English | DB의 사용자_설정 UPDATE |
| 12 | 로그아웃 → 로그인 | 토큰 삭제, 자동 로그인 안 됨 |

> ⚠️ **11단계 사전 확인**: `GET /api/settings`, `PATCH /api/settings` 엔드포인트가 백엔드에 구현되어 있어야 합니다. 404가 뜨면 `backend/app/api/settings.py` 생성 및 `main.py`에 라우터 등록 필요.

**12단계가 모두 통과되면 1·2주차 MVP 인수 완료!**
