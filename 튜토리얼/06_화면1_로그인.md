# 06. 화면 1 — 로그인 (Clean Architecture 표준 예시)

> **이 단계가 끝나면**: 로그인 화면이 백엔드와 연동되어 동작합니다. 검사원 ID/이름/비밀번호 입력 → 백엔드 인증 성공 시 토큰 저장 → 탱크 위치 선택 화면으로 이동.
>
> **예상 시간**: 4시간
>
> **중요도**: ★★★★★ — **이후 모든 화면이 이 패턴을 복제합니다.** 이 단계만 완벽히 이해하면 나머지 화면은 빠르게 만들 수 있습니다.

> **참조 명세서**:
> - `frontend/design/PAGE/01_로그인/COMPONENT.md` (UI 라벨·문구 원문)
> - `화면_명세서.md` 화면 1
> - `API_SPEC.md` AUTH-001
> - `frontend/design/CLAUDE.md` §2 (Clean Architecture 규칙)

## 1. 이 화면의 Clean Architecture 구조

```
features/auth/
├── domain/                      ← 외부 의존성 0, 순수 Dart
│   ├── entities/
│   │   └── inspector.dart       (검사원 도메인 객체)
│   ├── repositories/
│   │   └── auth_repository.dart (추상 클래스)
│   └── usecases/
│       └── login_usecase.dart   (비즈니스 동작 단위)
│
├── data/                        ← 외부 시스템 연결
│   ├── datasources/
│   │   └── auth_remote_data_source.dart  (Dio 호출)
│   ├── models/
│   │   ├── login_request_dto.dart
│   │   └── login_response_dto.dart       (JSON ↔ Dart)
│   └── repositories/
│       └── auth_repository_impl.dart      (domain 추상 구현)
│
└── presentation/                ← UI
    ├── providers/
    │   └── auth_providers.dart  (Riverpod Provider 묶음)
    ├── notifiers/
    │   └── login_notifier.dart  (StateNotifier + freezed State)
    └── screens/
        └── auth_screen.dart     (로그인 UI)
```

흐름:
```
UI 위젯 (auth_screen)
  → ref.read(loginNotifierProvider.notifier).login(...)
  → LoginNotifier
  → LoginUseCase.execute(...)
  → AuthRepository (추상)
       ↑ Riverpod이 AuthRepositoryImpl 주입
  → AuthRepositoryImpl
  → AuthRemoteDataSource.login(...)  ← Dio 호출
  ↓ 응답이 거꾸로 돌아오면서
  ↓ DTO → Entity → State 변환
UI 위젯이 새 State를 받아 리렌더링
```

```dart
class LoginUseCase {
  final AuthRepository repo;
  LoginUseCase(this.repo);

  Future<LoginResult> execute({
    required int inspectorId,
    required String name,
    required String password,
  }) {
    // 비즈니스 정책이 있다면 여기에 (현재는 단순 위임)
```

```dart
class AuthRemoteDataSource {

  Future<LoginResponseDto> login(LoginRequestDto req) async {

    await dio.post('/auth/logout');
  }
}
```

### 3-5. `data/repositories/auth_repository_impl.dart`

```dart
import 'package:dio/dio.dart';

import '../../../../core/error/failure.dart';
import '../../../../core/storage/token_storage.dart';
import '../../domain/repositories/auth_repository.dart';
import '../datasources/auth_remote_data_source.dart';
import '../models/login_request_dto.dart';

class AuthRepositoryImpl implements AuthRepository {
  final AuthRemoteDataSource remote;
  final TokenStorage tokenStorage;

  AuthRepositoryImpl({required this.remote, required this.tokenStorage});

  @override
  Future<LoginResult> login({
    required int inspectorId,
    required String name,
    required String password,
  }) async {
    try {
      final dto = await remote.login(LoginRequestDto(
        inspector_id: inspectorId,
        name: name,
        password: password,
      ));
      final result = dto.toResult();
      await tokenStorage.saveTokens(
        accessToken: result.accessToken,
        refreshToken: result.refreshToken,
        inspectorId: result.inspector.inspectorId,
      );
      return result;
    } on DioException catch (e) {
      throw _mapError(e);
    }
  }

  @override
  Future<void> logout() async {
    try {
      await remote.logout();
    } catch (_) {
      // best-effort: 서버 실패해도 로컬 토큰은 지움
    } finally {
      await tokenStorage.clear();
    }
  }

  Failure _mapError(DioException e) {
    final status = e.response?.statusCode;
    final data = e.response?.data;
    final Map? detail = data is Map ? data['detail'] as Map? : null;
    final errorCode = detail?['error'];

    if (status == 401 && errorCode == 'AUTH_FAILED') {
      return const AuthFailure();
    }
    if (status == 403 && errorCode == 'INACTIVE_ACCOUNT') {
      return const InactiveAccountFailure();
    }
    if (e.type == DioExceptionType.connectionError || e.type == DioExceptionType.connectionTimeout) {
      return const NetworkFailure();
    }
    if (status != null && status >= 500) {
      return const ServerFailure();
    }
    return const UnknownFailure();
  }
}
```

### 4-2. `presentation/notifiers/login_notifier.dart`

```dart
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:freezed_annotation/freezed_annotation.dart';

import '../../../../core/error/failure.dart';
import '../../domain/entities/inspector.dart';
import '../providers/auth_providers.dart';

part 'login_notifier.freezed.dart';

@freezed
class LoginState with _$LoginState {
  const factory LoginState({
    @Default(false) bool isLoading,
    String? errorMessage,
    Inspector? inspector,
  }) = _LoginState;
}

class LoginNotifier extends StateNotifier<LoginState> {
  final Ref ref;
  LoginNotifier(this.ref) : super(const LoginState());

  Future<bool> login({
    required int inspectorId,
    required String name,
    required String password,
  }) async {
    state = state.copyWith(isLoading: true, errorMessage: null);

    try {
      final result = await ref.read(loginUseCaseProvider).execute(
            inspectorId: inspectorId,
            name: name,
            password: password,
          );
      state = state.copyWith(isLoading: false, inspector: result.inspector);
      ref.read(currentInspectorProvider.notifier).state = result.inspector;
      return true;
    } on Failure catch (f) {
      state = state.copyWith(isLoading: false, errorMessage: f.message);
      return false;
    } catch (e) {
      state = state.copyWith(isLoading: false, errorMessage: '알 수 없는 오류가 발생했습니다.');
      return false;
    }
  }

  void clearError() {
    state = state.copyWith(errorMessage: null);
  }
}

final loginNotifierProvider = StateNotifierProvider<LoginNotifier, LoginState>(
  (ref) => LoginNotifier(ref),
);
```

## 5. 라우터의 인증 가드 추가

`lib/core/router/app_router.dart`의 GoRouter 정의에 `redirect` 콜백 추가:

```dart
import '../../core/storage/token_storage.dart';
import '../../features/auth/presentation/providers/auth_providers.dart';

// currentInspectorProvider 변화를 GoRouter에 전달하는 브릿지
class _RouterNotifier extends ChangeNotifier {
  _RouterNotifier(Ref ref) {
    ref.listen(currentInspectorProvider, (_, __) => notifyListeners());
  }
}

final appRouterProvider = Provider<GoRouter>((ref) {
  final tokenStorage = ref.watch(tokenStorageProvider);
  final notifier = _RouterNotifier(ref);

  return GoRouter(
    initialLocation: AppRoutes.login,
    debugLogDiagnostics: true,
    refreshListenable: notifier,  // Inspector 상태 변화 시 redirect 재평가
    redirect: (context, state) async {
      final token = await tokenStorage.getAccessToken();
      final isAuthed = token != null;
      final goingToLogin = state.matchedLocation == AppRoutes.login;

      if (!isAuthed && !goingToLogin) {
        return AppRoutes.login;  // 미인증 → 로그인으로
      }
      if (isAuthed && goingToLogin) {
        return AppRoutes.tankLocation;  // 인증된 사용자가 다시 로그인 페이지로? → 탱크 선택으로
      }
      return null;  // 통과
    },
    routes: [...],  // 기존 그대로
  );
});
```

> 💡 **`refreshListenable`이 하는 일**: 로그인 시 `currentInspectorProvider`에 Inspector가 설정되거나, 로그아웃 시 null로 초기화되면 `_RouterNotifier`가 GoRouter에 알려 redirect를 재평가합니다. 이 덕분에 `context.go('/login')`을 명시적으로 호출하지 않아도 인증 상태에 따라 화면이 자동으로 전환됩니다.

## 6. 동작 확인

1. 백엔드 실행 중인지 확인 (`uvicorn` 창)
2. 에뮬레이터 시작 후 `flutter run`
3. 로그인 화면이 명세서 그대로 표시되는지 확인:
   - 상단 factory 아이콘 + "Industrial Smart Inspection"
   - 검사원 ID, 성함, 비밀번호 3개 입력란
   - 아래 "로그인" 버튼
   - 맨 아래 저작권

4. **테스트 케이스**:
   - 빈 입력으로 "로그인" → 각 필드 빨간 에러 메시지
   - ID=1, 이름=김검사, 비번=`wrong` → "ID 또는 비밀번호가 올바르지 않습니다." 빨간 스낵바
   - ID=1, 이름=김검사, 비번=`test1234` → **성공** → 탱크 위치 선택 화면(아직 placeholder)으로 이동

5. **재실행 시 자동 로그인 확인**: 앱을 완전히 종료 → 다시 실행 → 로그인 화면이 아니라 `/tank-location`으로 바로 이동되어야 함 (토큰이 남아 있으므로)

```dart
        child: const InspectionApp(),

    );
    await tester.pumpAndSettle();

    // 로그인 화면 핵심 요소 확인
    expect(find.text('로그인'), findsOneWidget);
    expect(find.text('검사원 ID'), findsOneWidget);
  });
}
```

**핵심 포인트:**
- `flutter/material.dart`는 이 파일에서 직접 사용하지 않으므로 import하지 않습니다 (`find.text` 등은 `flutter_test`에 포함)
- `FlutterSecureStorage`는 테스트 환경에서 플랫폼 채널이 없어 오류 발생 → `_FakeTokenStorage`로 Provider를 override
- `pumpAndSettle()`로 GoRouter의 비동기 redirect까지 완료 대기
- token=null이므로 redirect 조건 모두 불충족 → `/login` 유지

## 7. 이 패턴을 기억하세요

다음 화면들도 같은 구조입니다:

```
features/<feature>/
├── domain/
│   ├── entities/         ← 도메인 객체
│   ├── repositories/     ← 추상 인터페이스
│   └── usecases/         ← (선택) 비즈니스 동작
├── data/
│   ├── datasources/      ← Dio·sqflite 호출
│   ├── models/           ← DTO (freezed + JSON)
│   └── repositories/     ← 구현체
└── presentation/
    ├── providers/        ← Riverpod Provider 정의
    ├── notifiers/        ← StateNotifier + freezed State
    └── screens/          ← UI
```

핵심 흐름:
1. **DTO 정의** (`@freezed` + `@JsonSerializable`)
2. **DataSource**가 Dio 호출
3. **RepositoryImpl**이 DTO → Entity 변환 + 에러 매핑
4. **UseCase**가 Repository를 호출 (단순 위임이라도)
5. **Notifier**가 UseCase 호출 + State 변경
6. **Screen**이 Notifier의 State를 watch + 메서드 호출

## 자주 발생하는 오류와 해결

### `LoginResponseDto.fromJson` 컴파일 에러
- `dart run build_runner build --delete-conflicting-outputs` 실행했는지 확인
- 빈 줄 또는 누락된 import가 있는지 확인

### freezed factory 컴파일 에러 ("missing default constructor")
- freezed 클래스 안에 메서드를 두려면 `const ClassName._();` 같은 private constructor 필요

### JSON 키 이름 mismatch
- DTO 필드명이 백엔드 응답과 정확히 일치해야 함 (`inspector_id`, `access_token` 등)
- 백엔드 응답을 Swagger UI에서 한 번 더 확인

### 라우터 redirect 무한 루프
- 인증된 상태에서 `/login`으로 가면 `/tank-location`으로 redirect → 그게 또 trigger
- `goingToLogin && isAuthed`일 때만 redirect하도록 조건 확실히

### 토큰이 저장 안 됨 (재시작 시 또 로그인 화면)
- `flutter_secure_storage` 권한 문제 가능. 폰의 보안 잠금 화면이 설정되어 있어야 키스토어 사용 가능 (에뮬레이터는 자동)
