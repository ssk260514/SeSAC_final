# 01. 개발 환경 셋업 (Windows 11 기준)

> **이 단계가 끝나면**: Flutter SDK + VS Code + 안드로이드 에뮬레이터 + Python + Docker가 모두 깔린 상태가 됩니다. 빈 Flutter 앱이 에뮬레이터에서 "Hello World"를 보여주면 성공.
>
> **예상 시간**: 2~3시간 (다운로드 대기 시간 포함)

> ⚠️ **이 단계는 한 번만 합니다.** 컴퓨터가 바뀌거나 새 팀원이 합류할 때만 다시 합니다.

---

## 사전 준비

- Windows 11 (Home/Pro 모두 가능)
- 디스크 여유 공간 **30GB 이상** (Flutter SDK + Android SDK + 에뮬레이터 이미지 등)
- 안정적인 인터넷 (총 다운로드 약 10GB)
- 관리자 권한 계정

---

## 1. Git 설치

가장 먼저 Git을 깝니다. 코드 버전 관리뿐 아니라 Flutter 자체가 내부적으로 Git을 씁니다.

1. https://git-scm.com/download/win 접속 → "Standalone Installer" 64-bit 다운로드
2. 설치 시 옵션은 **전부 기본값**으로 두고 Next 연타
3. 설치 완료 후 **PowerShell**을 새로 열고 (이미 열린 창은 환경변수가 갱신 안 됨):
   ```powershell
   git --version
   ```
   `git version 2.xx.x.windows.x` 같이 나오면 성공.

> 💡 **왜?** Flutter SDK가 내부적으로 `git fetch`로 업데이트를 가져옵니다. Git이 없으면 `flutter upgrade`가 실패합니다.

## 2. Flutter SDK 설치

### 2-1. SDK 다운로드

1. https://docs.flutter.dev/get-started/install/windows 접속
2. "flutter_windows_x.xx.x-stable.zip" 파일 다운로드 (약 1GB)
3. 압축 해제 위치를 정합니다. **공백·한글이 없는 경로**를 추천합니다:
   ```
   C:\dev\flutter\
   ```
   - ❌ `C:\Program Files\flutter\` — 공백 때문에 일부 도구가 깨짐
   - ❌ `C:\Users\홍길동\flutter\` — 한글이 PATH에 들어가면 깨짐
   - ✅ `C:\dev\flutter\`

4. 압축을 풀면 `C:\dev\flutter\bin\flutter.bat` 파일이 보여야 합니다.

### 2-2. 환경 변수에 PATH 추가

1. Windows 검색창에 "환경 변수" 입력 → "시스템 환경 변수 편집" 클릭
2. 하단 **[환경 변수(N)...]** 클릭
3. 상단 "사용자 변수" → **Path** 선택 → **편집**
4. **새로 만들기** → `C:\dev\flutter\bin` 입력
5. 모든 창에서 **확인** 클릭

### 2-3. 확인

**PowerShell을 새로 열고** (환경 변수 적용을 위해):
```powershell
flutter --version
```

다음과 비슷한 출력이 나오면 성공:
```
Flutter 3.24.x • channel stable • https://github.com/flutter/flutter.git
Framework • revision xxxxxxxxxx
Engine • revision xxxxxxxxxx
Tools • Dart 3.5.x • DevTools 2.xx.x
```

### 2-4. flutter doctor 첫 실행

```powershell
flutter doctor
```

이 시점에는 빨간 X가 잔뜩 나옵니다. **정상입니다.** 차근차근 다음 단계들에서 하나씩 해결합니다.

```
[✓] Flutter (Channel stable, ...)
[✗] Windows Version (..)
[✗] Android toolchain
[✗] Chrome
[✗] Visual Studio
[✗] Android Studio
[✗] VS Code
[✓] Connected device
[✗] Network resources
```

> 💡 **Windows Version**이 ✗로 나오는 경우 무시해도 됩니다 (Windows 11이면 충족).

## 3. VS Code 설치

1. https://code.visualstudio.com 접속 → **Download for Windows** (User Installer 64-bit)
2. 설치 시 다음 옵션 모두 체크:
   - ☑ "PATH에 추가"
   - ☑ "Code로 열기" 컨텍스트 메뉴 추가 (파일·폴더 양쪽)
3. 설치 완료 → VS Code 실행

### 3-1. 필수 확장 4종 설치

VS Code 좌측 사이드바의 **확장(Extensions)** 아이콘(네모 4개) 클릭 → 다음을 검색해서 설치:

| 확장 이름 | 만든이 | 용도 |
|---|---|---|
| **Flutter** | Dart Code | Flutter 개발 핵심. Dart도 자동 설치됨 |
| **Dart** | Dart Code | (Flutter 설치 시 자동 설치) |
| **Error Lens** | Alexander | 에러를 코드 줄 옆에 인라인 표시 (필수) |
| **GitLens** | GitKraken | Git 히스토리·blame을 코드 안에서 |

### 3-2. VS Code 설정 (선택)

`Ctrl + ,` → 검색창에 다음 입력 → 토글:

- `editor.formatOnSave`: ☑ 켜기 (저장 시 자동 포맷)
- `dart.previewFlutterUiGuides`: ☑ 켜기 (위젯 트리 시각화)

---

## 4. Android Studio 설치 (SDK + 에뮬레이터 용도)

> ⚠️ **우리는 VS Code로 코딩합니다.** Android Studio는 단지 Android SDK와 에뮬레이터 도구를 받기 위한 용도입니다.

### 4-1. 설치

1. https://developer.android.com/studio 접속 → **Download Android Studio** (약 1GB)
2. 설치 시 모든 옵션 **기본값**으로 진행
3. 처음 실행 시 "Standard" 셋업 → **Next** 연타. SDK 자동 다운로드 (약 4GB)

### 4-2. SDK 라이선스 동의

설치 끝나면 PowerShell에서:
```powershell
flutter doctor --android-licenses
```

라이선스 동의를 묻는 프롬프트가 여러 개 뜹니다. 매번 **y** + Enter.

### 4-3. 안드로이드 가상 기기(AVD) 만들기

1. Android Studio 시작 화면 → 우하단 **More Actions** → **Virtual Device Manager**
2. **Create Device** 클릭
3. Phone 카테고리에서 **Pixel 7** 선택 → **Next**
4. System Image:
   - **API 34 (Android 14)** 선택 — 옆에 **Download** 글자가 보이면 클릭 (약 1GB)
   - 다운로드 완료 후 **Next**
5. AVD Name은 그대로, **Finish**
6. 만들어진 가상 기기 옆의 ▶️ 재생 버튼으로 실행 → 안드로이드 화면이 뜨면 성공

> 💡 **에뮬레이터가 너무 느리다면** Intel HAXM 또는 Hyper-V 가속을 활성화해야 합니다. BIOS에서 Intel VT-x 또는 AMD-V를 켜야 할 수 있습니다. 실기기 USB 디버깅으로 대체해도 OK.

## 5. 실기기 USB 디버깅 (선택, 카메라 테스트용)

에뮬레이터의 카메라는 **가짜 영상**이라 09번 단계(카메라 촬영)에서는 실기기를 쓰는 게 좋습니다.

1. 안드로이드 폰에서 **설정** → **휴대전화 정보** → **빌드 번호 7번 탭** → 개발자 옵션 활성화
2. **설정** → **개발자 옵션** → **USB 디버깅** ON
3. USB로 PC에 연결 → 폰 화면에 뜨는 "USB 디버깅 허용" 다이얼로그 → 허용
4. PowerShell에서:
   ```powershell
   flutter devices
   ```
   본인 폰 모델명이 나오면 성공.

> ⚠️ 폰 제조사별 USB 드라이버가 필요할 수 있습니다 (삼성: Samsung USB Driver, LG: LG United Mobile Driver 등). 안 잡히면 제조사 사이트에서 다운로드.

## 6. Python 3.11 설치 (백엔드용)

### 6-1. 설치

1. https://www.python.org/downloads/windows/ → **Python 3.11.x** (3.12, 3.13이 아닌 **3.11**) 다운로드
2. 설치 시 **반드시** 첫 화면에서:
   - ☑ **Add python.exe to PATH** 체크
   - **Install Now** 클릭

### 6-2. 확인

PowerShell 새로 열고:
```powershell
python --version
pip --version
```
각각 `Python 3.11.x`, `pip 24.x.x ...`가 나오면 성공.

> 💡 **왜 3.11?** 3.12·3.13은 일부 ML 라이브러리(특히 torch, tflite-runtime)가 아직 정식 지원하지 않을 수 있습니다. 안전한 LTS인 3.11을 씁니다.

## 7. Docker Desktop 설치

PostgreSQL·pgvector를 컨테이너로 띄우기 위해 필요합니다.

### 7-1. WSL2 활성화 (Docker 의존성)

PowerShell을 **관리자 권한**으로 열고:
```powershell
wsl --install
```

설치 후 **재부팅**.

재부팅 후 다시 PowerShell:
```powershell
wsl --status
```
"기본 버전: 2"가 나오면 OK.

### 7-2. Docker Desktop 설치

1. https://www.docker.com/products/docker-desktop/ → **Download for Windows** (약 700MB)
2. 설치 시:
   - ☑ "Use WSL 2 instead of Hyper-V" (기본 체크됨)
3. 설치 후 재부팅
4. Docker Desktop 실행 → 로그인 건너뛰기 가능 → 우하단에 🐳 아이콘이 안정되면 OK

### 7-3. 확인

PowerShell에서:
```powershell
docker --version
docker run --rm hello-world
```

`Hello from Docker!` 메시지가 나오면 성공.

## 9. 첫 Flutter 앱으로 검증

지금까지 설치한 것들이 잘 묶여 있는지 빈 Flutter 앱으로 확인합니다.

### 9-1. 폴더 생성

```powershell
mkdir C:\dev\test_app
cd C:\dev\test_app
flutter create hello_app
cd hello_app
```

### 9-2. 에뮬레이터에서 실행

1. **에뮬레이터 또는 실기기를 먼저 켭니다.**
2. VS Code에서 `C:\dev\test_app\hello_app` 폴더를 엽니다 (`File → Open Folder`).
3. `lib/main.dart` 파일을 열고 우하단 상태 바에서 디바이스가 잡혔는지 확인 (예: `Pixel 7 API 34`)
4. **F5** 또는 메뉴에서 **Run → Start Debugging**
5. 첫 빌드는 **3~5분** 걸립니다 (Gradle이 의존성을 다운로드).
6. 에뮬레이터에 카운터 앱이 뜨면 성공. 우하단 ➕ 버튼으로 카운터가 증가하면 정상.

### 9-3. Hot Reload 체험

`lib/main.dart`에서 텍스트를 아무거나 바꿔서 저장하세요 (예: `'You have pushed...'` → `'Hello World'`). VS Code 하단 콘솔에 `Reloaded` 메시지가 나오면서 **즉시** 화면이 갱신됩니다. 이게 Flutter의 강력함.

## 자주 발생하는 오류와 해결

### "flutter: 명령을 찾을 수 없음"
- PowerShell을 **새로 열었는지** 확인. 환경 변수 추가 후에는 새 창에서만 적용됨
- `echo $env:PATH` 로 `C:\dev\flutter\bin`이 포함됐는지 확인

### "Unable to locate Android SDK"
- Android Studio를 한 번 실행하고 "Standard" 셋업을 끝까지 완료했는지 확인
- `flutter config --android-sdk "C:\Users\<유저명>\AppData\Local\Android\Sdk"` 로 수동 지정

### "Android license status unknown"
- `flutter doctor --android-licenses` 실행 후 모든 프롬프트에 `y` 입력

### Gradle 첫 빌드가 5분 넘게 멈춤
- 정상입니다. **인내심이 필요**합니다. 다운로드 중인지 `gradle-x.x.x-bin.zip` 파일이 `%USERPROFILE%\.gradle\` 아래에 만들어지고 있는지 확인하세요.

### 에뮬레이터가 너무 느림
1. BIOS에서 가상화(VT-x/AMD-V) 활성화
2. Docker Desktop이 켜져 있으면 메모리 경합 발생. 일시 종료
3. AVD 설정에서 RAM을 1024 → 2048MB로 늘리기

### Docker `wsl --install`이 안 됨
- Windows 업데이트가 최신인지 확인
- 회사 IT 정책으로 막혀 있을 수 있음 → IT팀에 문의

### 폰이 `flutter devices`에 안 잡힘
1. USB 케이블을 데이터 전용이 아닌 충전 전용을 쓰고 있는 건 아닌지 확인
2. 폰에서 "USB 디버깅 허용" 다이얼로그를 놓쳤을 수 있음. USB 뺐다 다시 꽂으면 다시 뜸
3. 제조사 USB 드라이버 설치 필요할 수 있음
