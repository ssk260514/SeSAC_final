> **이 단계가 끝나면**: 양품 사진 중 10%가 자동으로 S3에 비동기 업로드되어 **단일 통합 모델 재학습용 데이터셋**이 누적되고, 오프라인 상태에서 검사한 결과가 네트워크 복구 시 50건씩 청크로 자동 동기화됩니다. 멱등성 키로 중복 INSERT가 방지됩니다.

## 1. 왜 양품 샘플링 + 오프라인 큐인가

### 1-1. 양품 샘플링 (INFER-004)
- 양품은 단말기에서 자동 종결되므로 서버는 양품 사진을 못 봅니다 → **학습 데이터 편향 위험** (모델이 양품을 점점 모르게 됨)
- 모든 양품을 보내면 통신량 폭증·S3 비용 증가
- **타협**: 양품 중 **무작위 10%만** 비동기 업로드 → 학습 데이터 일부 확보 + 통신·비용 절약

### 1-2. 오프라인 큐 (INFER-005)
- 탱크 내부·지하 등 통신 음영 지역에서 검사 가능해야 함
- 단말 추론 결과는 sqflite 로컬 큐에 저장 → 네트워크 복구 시 **50건씩 청크 단위로 일괄 업로드**
- 같은 요청이 두 번 전송되어도 DB에 중복 INSERT 되지 않도록 **멱등성 키**(`client_request_id` UUID v4) 사용

```
양품 분류 (단말, 신뢰도≥0.85)
    │
    ├─ 10% 확률 → POST /api/inspect/sample-upload (INFER-004)
    │             ↓ 사내 S3에 비동기 업로드 (앱은 안 기다림)
    └─ 90%      → 단말에서 종결 (서버 호출 없음)

불량 분류 또는 신뢰도<0.85
    │
    ├─ ONLINE   → POST /api/inspect (INFER-002, 서버 정밀 분석)
    └─ OFFLINE  → sqflite 큐에 저장 + client_request_id 부여
                  ↓ ONLINE 복귀
                  → POST /api/inspect/offline-batch (INFER-005, 50건 청크)
```

> 💡 13번에서 만든 오프라인 큐 골격을 **정식 API로 정식화**하는 단계입니다. 13번에서는 단순 더미 처리였다면, 17번에서는 멱등성 키 + 청크 전송 + 캐시 기반 중복 차단으로 운영 수준으로 끌어올립니다.

## 3. 백엔드 — INFER-004 양품 샘플 업로드

### 3-1. `backend/app/api/inspect.py` (기존 파일에 추가 또는 분리)

```python
import uuid
from datetime import datetime

import aioboto3
from fastapi import APIRouter, Depends, File, Form, HTTPException, UploadFile
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import text

from app.core.db import get_db
from app.core.config import settings
from app.api.deps import get_current_inspector_id

router = APIRouter()


@router.post("/inspect/sample-upload")
async def sample_upload(
    image: UploadFile = File(...),
    session_id: int = Form(...),
    process_id: int = Form(...),         # 위치/RAG 메타용 (모델 선택 무관 — 단일 모델)
    defect_type: str = Form(...),        # 단말이 분류한 양품 클래스 (예: "표면양품-도장")
    confidence: float = Form(...),
    captured_at: str = Form(...),        # ISO 8601
    inspector_id: int = Depends(get_current_inspector_id),
    db: AsyncSession = Depends(get_db),
):
    """양품 사진 10% 샘플 — S3 비동기 업로드 + 검사_이미지 메타 INSERT (INFER-004).

    단일 모델 재학습용 데이터셋 누적. 본 API는 양품 클래스명만 받습니다."""

    if "양품" not in defect_type:
        raise HTTPException(status_code=400, detail={"error": "NOT_A_PASS_CLASS"})

    # 1) S3 키 생성 — 클래스/일자별 폴더 (학습팀이 ETL 시 활용)
    today = datetime.utcnow().strftime("%Y/%m/%d")
    s3_key = f"samples/{today}/{defect_type}/{uuid.uuid4().hex}.jpg"

    # 2) S3 비동기 업로드 (앱에는 즉시 200 반환 — fire-and-forget 안 함, but background)
    session_factory = aioboto3.Session()
    async with session_factory.client(
        "s3",
        aws_access_key_id=settings.AWS_ACCESS_KEY_ID,
        aws_secret_access_key=settings.AWS_SECRET_ACCESS_KEY,
        region_name=settings.AWS_REGION,
    ) as s3:
        data = await image.read()
        await s3.put_object(Bucket=settings.S3_BUCKET, Key=s3_key, Body=data,
                            ContentType="image/jpeg")

    # 3) 검사_이미지에 메타 기록 (검사_결과는 단말 종결이라 INSERT 안 함 — 양품 샘플 전용)
    await db.execute(text("""
        INSERT INTO 검사_이미지 (세션_ID, 이미지_경로, 촬영_일시)
        VALUES (:sid, :path, :ts)
    """), {"sid": session_id, "path": f"s3://{settings.S3_BUCKET}/{s3_key}",
            "ts": captured_at})
    await db.commit()

    return {"uploaded": True, "s3_key": s3_key}
```

> 💡 **왜 검사_결과는 INSERT 하지 않는가**: 양품은 단말기에서 이미 자동 종결됐고, 검사_결과는 이미 단말 결과 1행으로 저장되어 있습니다. 본 API는 **재학습용 이미지만** S3에 누적하는 별도 흐름입니다 (`기능_명세서.md` §4-3 INFER-004).

`pyproject.toml` 또는 `requirements.txt` 에 추가:

```

```

## 4. 백엔드 — INFER-005 오프라인 배치

### 4-1. 멱등성 키 캐시

같은 `client_request_id` 가 두 번 들어오면 두 번째는 무시. **간단한 DB 테이블**(또는 Redis)로 구현합니다.

`backend/sql/003_idempotency.sql` (신규):

```sql
CREATE TABLE 멱등성_요청 (
    client_request_id UUID PRIMARY KEY,
    검사원_ID        INT NOT NULL REFERENCES 검사원(검사원_ID),
    처리_결과         JSONB,
    생성_일시         TIMESTAMP NOT NULL DEFAULT NOW()
);

-- 30일 이상 된 항목은 정리 (운영자 cron)
CREATE INDEX idx_idempotency_created ON 멱등성_요청(생성_일시);
```

> 💡 본 테이블은 12개 핵심 테이블에 포함되지 않은 **운영 유틸** 입니다 (`schema_diagram.md` 외). 30일 보관 후 cron 정리.

### 4-2. `backend/app/api/inspect.py` 에 배치 엔드포인트 추가

```python
from typing import List
from pydantic import BaseModel, Field


class OfflineItem(BaseModel):
    client_request_id: str = Field(..., description="UUID v4 — 멱등성 키")
    session_id: int
    process_id: int                  # 위치/RAG 메타 (모델 선택 무관)
    defect_type: str
    confidence: float
    captured_at: str                 # ISO 8601
    image_base64: str                # 작은 이미지 (224x224 압축 권장)


@router.post("/inspect/offline-batch")
async def offline_batch(
    items: List[OfflineItem],
    inspector_id: int = Depends(get_current_inspector_id),
    db: AsyncSession = Depends(get_db),
):
    """오프라인 큐 배치 업로드 (INFER-005). 50건 청크 권장. 멱등성 키로 중복 차단."""
    if len(items) > 50:
        raise HTTPException(status_code=413, detail={"error": "BATCH_TOO_LARGE"})

    accepted = []
    skipped_duplicates = []
    for item in items:
        # 1) 멱등성 체크
        existing = (await db.execute(text("""
            SELECT 처리_결과 FROM 멱등성_요청 WHERE client_request_id = :rid
        """), {"rid": item.client_request_id})).first()
        if existing is not None:
            skipped_duplicates.append(item.client_request_id)
            continue

        # 2) 처리: 검사_이미지·검사_결과 INSERT (단말 결과 그대로, 추론_위치='단말')
        is_defect = "양품" not in item.defect_type
        # ... 검사_이미지·검사_결과 INSERT (생략, 09번/13번 흐름 동일)

        # 3) 멱등성 기록 (재시도 들어와도 중복 차단)
        await db.execute(text("""
            INSERT INTO 멱등성_요청 (client_request_id, 검사원_ID, 처리_결과)
            VALUES (:rid, :iid, :res)
        """), {"rid": item.client_request_id, "iid": inspector_id,
                "res": '{"status":"ok"}'})
        accepted.append(item.client_request_id)

    await db.commit()
    return {"accepted": accepted, "skipped_duplicates": skipped_duplicates}
```

> 💡 **왜 50건 제한**: 한 요청이 너무 크면 네트워크 단절 시 처음부터 재시도해야 합니다. 50건씩 끊으면 실패해도 그 50건만 재전송하면 되어 효율적입니다.

### 5-2. `lib/features/capture/data/local/offline_queue_service.dart` (13번 골격 확장)

```dart
import 'package:connectivity_plus/connectivity_plus.dart';
import 'package:dio/dio.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:sqflite/sqflite.dart';
import 'package:uuid/uuid.dart';

import '../../../../core/network/dio_client.dart';

class OfflineQueueService {
  OfflineQueueService(this.db, this.dio);
  final Database db;
  final Dio dio;
  static const _chunkSize = 50; // INFER-005 권장값

  /// 단말 추론 결과 enqueue — 통신 음영에서 호출
  Future<void> enqueue({
    required int sessionId,
    required int processId,
    required String defectType,
    required double confidence,
    required String capturedAtIso,
    required String imageBase64,
  }) async {
    await db.insert('offline_queue', {
      'client_request_id': const Uuid().v4(),
      'session_id': sessionId,
      'process_id': processId,
      'defect_type': defectType,        // 30클래스 중 하나
      'confidence': confidence,
      'captured_at': capturedAtIso,
      'image_base64': imageBase64,
      'created_at': DateTime.now().toIso8601String(),
    });
  }

  /// ONLINE 복귀 시 호출 — 50건 청크로 자동 flush
  Future<void> flush() async {
    while (true) {
      final rows = await db.query('offline_queue',
          orderBy: 'created_at ASC', limit: _chunkSize);
      if (rows.isEmpty) return;

      final items = rows.map((r) => {
            'client_request_id': r['client_request_id'],
            'session_id': r['session_id'],
            'process_id': r['process_id'],
            'defect_type': r['defect_type'],
            'confidence': r['confidence'],
            'captured_at': r['captured_at'],
            'image_base64': r['image_base64'],
          }).toList();

      try {
        final res = await dio.post('/inspect/offline-batch', data: items);
        final accepted = (res.data['accepted'] as List).cast<String>();
        final skipped = (res.data['skipped_duplicates'] as List).cast<String>();
        // accepted + skipped 둘 다 큐에서 삭제 (서버가 인지했음)
        for (final rid in [...accepted, ...skipped]) {
          await db.delete('offline_queue',
              where: 'client_request_id = ?', whereArgs: [rid]);
        }
      } on DioException {
        // 네트워크 재단절 — 다음 ONLINE 복귀까지 대기
        return;
      }
    }
  }
}

/// ONLINE 전환 감지 → 자동 flush
class OfflineSyncProvider {
  OfflineSyncProvider(this.service);
  final OfflineQueueService service;

  void start() {
    Connectivity().onConnectivityChanged.listen((events) async {
      final hasNet = events.any((e) => e != ConnectivityResult.none);
      if (hasNet) await service.flush();
    });
  }
}

final offlineSyncProvider = Provider<OfflineSyncProvider>((ref) {
  // db, dio 의존성 주입은 프로젝트의 기존 패턴(13번) 따름
  throw UnimplementedError('13번에서 작성한 dbProvider·dioProvider와 결선 필요');
});
```

## 6. 동작 확인

1. **양품 샘플링**: 양품 사진 100장을 연속 촬영 → 10번 안팎 (8~14건) `POST /api/inspect/sample-upload` 호출 → S3 버킷의 `samples/YYYY/MM/DD/{양품-클래스명}/{uuid}.jpg` 에 업로드 확인
2. **오프라인 큐**: 비행기 모드 → 불량 사진 60장 촬영 → sqflite `offline_queue` 60행 INSERT 확인
3. **자동 동기화**: 비행기 모드 해제 → `connectivity_plus` 가 ONLINE 감지 → `flush()` 가 50건 + 10건 = 2회 호출 → 큐가 비어짐
4. **멱등성**: 같은 `client_request_id` 두 번 전송 → 두 번째는 `skipped_duplicates` 에 들어감 → DB 검사_결과는 1행만
5. **단일 모델 정합**: 양품 클래스명이 `모델_레지스트리.클래스_라벨` 30개 중 하나(`'양품' in defect_type`)인지

```sql
-- S3 업로드 확인
SELECT 이미지_경로 FROM 검사_이미지 WHERE 이미지_경로 LIKE 's3://%/samples/%' LIMIT 10;

-- 멱등성 키 확인
SELECT count(*) FROM 멱등성_요청 WHERE 생성_일시 > now() - interval '1 hour';
```

## 자주 발생하는 오류와 해결

### S3 업로드 403 Forbidden
- IAM 정책에 `s3:PutObject` 권한이 있는지 확인
- 버킷 정책의 `Principal` 이 본 IAM 사용자를 포함하는지

### `offline-batch` 가 413 BATCH_TOO_LARGE
- 클라이언트가 50건 초과 전송. `_chunkSize` 가 50인지 확인

### 같은 결과가 검사 이력에 2번 표시
- `client_request_id` 생성 로직이 매번 새 UUID를 만드는지 확인. 재전송 시에도 **같은** UUID 사용 — `enqueue` 시점에 부여하고 큐에 보관

### `aioboto3` import 오류
- `pip install aioboto3` 또는 `requirements.txt` 갱신 후 재설치

### connectivity_plus가 ONLINE을 늦게 감지
- iOS는 connectivity 권한 필요 (Info.plist). Android 는 별도 권한 불필요
- 일부 단말은 셀룰러 → Wi-Fi 전환 시 이벤트가 두 번 발생. `flush()` 가 idempotent 하므로 OK
