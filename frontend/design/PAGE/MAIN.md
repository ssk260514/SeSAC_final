## 화면 흐름도 (Navigation Flow)

```
[화면 1: 로그인]
      │
      ▼
[화면 2: 탱크 유형 및 위치 선택]  ◄──────────┐
      │  1단계: 탱크 유형 (B/C) 선택           │
      │  2단계: 구역 + 세부 위치 선택           │ "위치 변경" 의도
      │  (선택값은 Riverpod 전역 상태 +        │ (화면 3 AppBar 아이콘)
      │   flutter_secure_storage 영속화)       │
      ▼                                         │
[메인 앱 — Bottom Navigation Bar 4탭]  ────────┘
 │
 ├─ [탭1] 화면 3: 대시보드 (홈)
 │         ├─ AppBar "위치 변경" 아이콘 → [화면 2] 재진입 (탱크/위치 변경)
 │         ├─ "검사 시작" → [화면 5: 카메라] (당일 세션 자동 생성, 1일 1세션)
 │         │                    │
 │         │                    셔터 → 카메라 유지 (백그라운드 추론)
 │         │                    │
 │         │                    "검사 이력" 썸네일 → [탭2]
 │         │
 │         └─ 최근 세션 이력 (최근 3건)
 │             ※ 최근 세션 이력 = 세션 완료된 결과만 표시
 │             
 │      
 │
 ├─ [탭2] 화면 4: 검사 이력 (당일 세션의 검사 항목, 1일 1세션)
 │         ├─ 필터: 전체 / 검사 완료 / 검사 미완료
 │         ├─ "검사 완료" 카드 클릭 → [화면 6: 결과 처리 (탭3)] (저장된 결과 열람/수정)
 │         ├─ "검사 미완료" 불량 카드 클릭 → [정밀 분석 모달 (Bottom Sheet)]
 │         │                                    │
 │         │                                    "결과 처리" → [탭3]
 │         ├─ "이어서 검사" 플로팅 버튼 → [화면 5: 카메라]
 │         └─ "세션 완료" 플로팅 버튼 (검사 미완료 0건일 때만 활성화)
 │               
 │               
 │
 ├─ [탭3] 화면 6: 결과 처리
 │         ├─ 기본: 읽기 전용 (저장된 결과 표시)
 │         ├─ "수정" 버튼 → 편집모드 활성화
 │         ├─ "저장" 버튼 → 확정 저장 → 검사 이력 탭으로 이동
 │         └─ 검사 이력 [탭2] - "검사 완료" 항목 클릭 시 언제든지 화면 6으로 재진입하여 저장된 결과를 열람하거나, 수정하여 재저장 가능
 │
 └─ [탭4] 화면 7: 설정
           ├─ 프로필, 알림 토글, 언어, 앱 버전
           └─ 로그아웃 → [화면 1]
```

## 공통 UI 요소

### Bottom Navigation Bar (4탭)

**적용 화면**: 대시보드, 검사 이력, 결과 처리, 설정
**비적용 화면**: 로그인, 탱크 유형 및 위치 선택, 카메라 (전체 화면 전용)

| 탭 # | 라벨 | 아이콘 (비활성/활성) | 타겟 화면 |
|---|---|---|---|
| 1 | 대시보드 | home / home (filled) | 화면 3 |
| 2 | 검사 이력 | history / history (filled) | 화면 4 |
| 3 | 결과 처리 | assignment / assignment (filled) | 화면 6 |
| 4 | 설정 | settings / settings (filled) | 화면 7 |

- 활성 탭: primary-container 배경 + on-primary-container 텍스트 + rounded-xl
- 비활성 탭: on-surface-variant 텍스트
- 높이: 80px, safe area 포함

### 모델 OTA 알림
- 앱 시작 시 모델 버전 체크 → 새 버전 있으면 백그라운드 다운로드
- 다운로드 완료 시: "새 AI 모델이 준비되었습니다. 다음 앱 시작 시 적용됩니다" 알림

### 에러 처리 공통
- 서버 연결 실패: "서버에 연결할 수 없습니다. 오프라인 모드로 전환합니다" 토스트
- JWT 만료: 자동으로 리프레시 토큰으로 갱신. 갱신 실패 시 로그인 화면으로 리다이렉트
- 서버 500 에러: "서버 오류가 발생했습니다. 잠시 후 다시 시도해주세요" 다이얼로그
