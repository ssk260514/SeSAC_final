# 디자인 시스템 명세서 — 선박 LNG탱크 부품 품질 검사 앱

## 문서 정보
- 기반 문서: `제품_정의서.md`, `기능_명세서.md`, `화면_명세서.md`
- 시각 참조: `화면html/code.html` (컴포넌트 구조·분위기 참조용)
- 디자인 토큰 체계: **Material Design 3 (역할 기반 토큰)**
- 대상 플랫폼: **Flutter (Android 우선)**
- 적용 범위: 1주차 MVP (라이트 모드만, 다크 모드는 Post-MVP)
- 브랜드: "Industrial Smart Inspection" / "LNG Safety Solutions Corp."

> 본 명세서의 **컴포넌트 사용 위치·문구·색상 지정은 화면명세서가 최종 기준**입니다. 토큰의 수치(컬러 HEX, 폰트 크기 등)는 본 명세서에서 신규 정의하며, code.html 의 수치는 채택하지 않습니다. 모든 화면 텍스트·뱃지 라벨·토스트 문구는 화면명세서 원문 그대로 사용합니다(요약본·해석본 금지).

## 1. 디자인 원칙 (Design Principles)

| # | 원칙 | 정의 | 적용 방식 |
|---|---|---|---|
| P1 | **현장 신뢰성 (Field Reliability)** | 오프라인·저신호·장갑 착용 환경에서도 끊김 없이 동작 | 큰 터치 타깃(56×56), 명시적 로딩·오프라인 상태 표시, 단순한 위계 |
| P2 | **명료한 판정 (Clear Verdict)** | 양품/불량 판정은 한눈에 식별 가능해야 함 | 색상바(좌측 4px) + 아이콘 + 라벨의 3중 표기. 색맹 사용자도 구별 가능 |
| P3 | **점진적 정보 공개 (Progressive Disclosure)** | 대시보드 요약 → 카드 리스트 → 상세 모달 의 3단계 깊이 | 카드는 핵심 정보만, 상세는 Bottom Sheet 모달에서 |
| P4 | **AI 투명성 (AI Transparency)** | AI 판정의 근거를 사용자가 확인할 수 있어야 함 | Grad-CAM 토글, Top-3 예측 노출, RAG 출처 매뉴얼 명시 |

> 이 4개 원칙은 모든 컴포넌트·패턴 선택의 기준입니다. 디자인 결정이 충돌할 때는 P1 → P2 → P3 → P4 순으로 우선합니다.

## 2. 컬러 시스템 (Color System)

### 2.1 컬러 시스템 개요
- **프레임워크**: Material Design 3 역할 기반 토큰
- **메인 톤**: 산업·안전 현장을 연상시키는 깊은 앰버(Deep Amber) 계열
- **Tertiary 팔레트 미사용**: 화면명세서가 tertiary 를 사용하지 않으므로 본 시스템에서도 정의하지 않음. AI/RAG/Grad-CAM 등 강조는 모두 primary 계열(`primary tint`) 로 통일

### 2.2 Primary 팔레트 (Amber — 브랜드 메인)
| 토큰 | HEX | 용도 |
|---|---|---|
| `primary` | `#B26A00` | 브랜드 텍스트·아이콘 강조, 액티브 아이콘 |
| `on-primary` | `#FFFFFF` | `primary` 위 텍스트 |
| `primary-container` | `#FFA726` | CTA 버튼 배경, 활성 탭 배경, Top-1 프로그레스 바, 셔터 버튼 내부, 카메라 가이드 보더 |
| `on-primary-container` | `#5C3500` | `primary-container` 위 텍스트·아이콘 |
| `primary-fixed` | `#FFDDB0` | 비활성·약한 강조 배경 |
| `primary-fixed-dim` | `#FFC078` | 호버·서브 강조 (웹 미리보기 한정) |

> `primary-container` 는 본 시스템에서 가장 빈번하게 사용되는 강조 색입니다. CTA·활성 탭·셔터·정보 칩·RAG 카드 틴트 모두 이 토큰 기반입니다.

### 2.4 Semantic Colors (의미 색상)

판정·상태를 즉시 식별할 수 있도록 의미 색상을 별도로 정의합니다. Material 3 표준 토큰이 아닌 항목(`success`, `warning`, `pending`, `info`)은 Flutter 의 `ThemeExtension` 으로 확장합니다.

| 역할 | 토큰 | HEX | 사용 위치 |
|---|---|---|---|
| **양품(Pass)** | `success` | `#2E7D32` | 검사 결과 카드 좌측 색상바(녹색), `check_circle` 아이콘 배경 |
| | `on-success` | `#FFFFFF` | success 위 텍스트 |
| | `success-container` | `#C8E6C9` | 성공 토스트 배경, "확정 저장됨" 뱃지 |
| | `on-success-container` | `#1B5E20` | |
| **불량(Fail)** | `error` | `#C62828` | 검사 결과 카드 좌측 색상바(빨강), `warning` 아이콘 배경, 로그아웃 버튼 보더·텍스트 |
| | `on-error` | `#FFFFFF` | |
| | `error-container` | `#FFDAD6` | 에러 토스트 배경 |
| | `on-error-container` | `#8B0000` | |
| **미완료(Pending)** | `pending` | `#877462` (= `outline`) | 검사 결과 카드 좌측 색상바(회색), "검사 미완료"·"분석 대기 중" 뱃지 |
| **정보(Info)** | `info` | `#4A5C70` (= `secondary`) | 정보 토스트 배경 |
| **경고(Warning)** | `warning` | `#ED6C02` | 비활성 공정·모델 미준비 경고 |

> **색맹 접근성**: 양품 그린(#2E7D32) 과 불량 레드(#C62828) 는 색상 단독이 아닌 **아이콘(`check_circle`/`warning`) 과 라벨("양품"/"불량") 을 항상 함께 표기**합니다 (원칙 P2).

## 3. 타이포그래피 (Typography)

### 3.1 폰트 패밀리

| 언어 | 폰트 | 굵기 | 출처 |
|---|---|---|---|
| **한글** (본문 기본) | **Pretendard** | 400 / 500 / 600 / 700 | [Pretendard Variable](https://github.com/orioncactus/pretendard) |
| **영문·숫자** | **Inter** | 400 / 500 / 600 / 700 | [Google Fonts](https://fonts.google.com/specimen/Inter) |
| Fallback | `system-ui`, `-apple-system`, `Roboto`, `sans-serif` | — | OS 기본 |

#### 폰트 단순화 옵션
Pretendard 는 한글·영문·숫자를 모두 자체 글리프로 지원하므로, **단일 폰트로 통일하는 옵션**을 권장합니다.
- **Option A (단순화 / 권장)**: Pretendard 단독. Inter 미적용.
- **Option B (페어링)**: 한글 Pretendard + 영문·숫자 Inter (`fontFamilyFallback` 으로 분기)

본 명세서의 토큰·예제는 Option A 기준으로 작성합니다.

### 3.2 타입 스케일

| 스타일 | 크기 / 라인높이 | 자간 | 굵기 | 용도 |
|---|---|---|---|---|
| `display` | 28 / 36 | -0.02em | 700 | 로그인 화면 브랜드명 |
| `h1` | 24 / 32 | -0.02em | 700 | 화면 타이틀 ("정밀 분석 결과", "검사 이력" 등) |
| `h2` | 20 / 28 | -0.01em | 600 | 카드 제목, 섹션 헤더 |
| `h3` | 18 / 26 | -0.01em | 600 | 서브 섹션 헤더 |
| `body-lg` | 16 / 24 | 0 | 400 | 본문 (기본) |
| `body-md` | 14 / 20 | 0 | 400 | 카드 본문, 보조 텍스트 (최소 본문 크기) |
| `label-bold` | 12 / 16 | 0.01em | 600 | 입력 라벨, 뱃지 텍스트, 탭 라벨 |
| `caption` | 11 / 14 | 0.02em | 400 | 타임스탬프, 메타데이터 |
| `code-data` | 14 / 20 | 0.05em | 500 (tabular-nums) | 신뢰도 % (예: "92.3%"), 앱 버전 (예: "v1.4.2"), 이미지 ID (예: "이미지 #102") |

> **현장 가독성**: 본문 최소 크기를 14px(`body-md`) 로 상향했습니다 (M3 기본 11~12px 보다 강화). 라인높이도 표준보다 약간 여유롭게 설정.

> **데이터 표기**: `code-data` 는 숫자가 흔들리지 않도록 **고정폭 숫자(`tabular-nums`)** 를 사용합니다. Pretendard 와 Inter 모두 지원.

## 4. 간격 & 레이아웃 (Spacing & Layout)

### 4.1 간격 스케일 (4px 베이스 그리드)

| 토큰 | 값 (px) | 용도 |
|---|---|---|
| `space-0` | 0 | reset |
| `space-1` | 4 | 인접 요소 간 최소 간격 (아이콘↔텍스트) |
| `space-2` | 8 | 라벨 ↔ 입력 |
| `space-3` | 12 | 작은 그룹 내 간격 |
| `space-4` | 16 | **기본** — 카드 패딩, 모바일 좌우 마진 |
| `space-5` | 20 | 중간 그룹 간격 |
| `space-6` | 24 | 섹션 간격 |
| `space-8` | 32 | 큰 섹션 분리 |
| `space-10` | 40 | 페이지 상하 여백 |

### 4.2 보더 반경 (Radius)

| 토큰 | 값 (px) | 용도 |
|---|---|---|
| `radius-sm` | 4 | 입력 내부 강조 (포커스 시 등) |
| `radius-md` | 8 | 작은 칩·뱃지 |
| `radius-lg` | 12 | **기본** — 카드·버튼·입력·드롭다운, 활성 탭 |
| `radius-xl` | 16 | 강조 카드 |
| `radius-2xl` | 24 | Bottom Sheet 모달 상단 라운딩 |
| `radius-full` | 9999 | 칩·아바타·세그먼티드 컨테이너 |

### 4.3 그림자 (Elevation)

| 토큰 | 정의 | 용도 |
|---|---|---|
| `elevation-0` | `none` | 평면 (구분선만 사용) |
| `elevation-1` | `0 1px 2px rgba(0,0,0,0.06)` | 카드·입력 (기본) |
| `elevation-2` | `0 2px 6px rgba(0,0,0,0.08)` | 액션 버튼·강조 카드 |
| `elevation-3` | `0 6px 16px rgba(0,0,0,0.12)` | 모달·플로팅 버튼 (FAB) |

### 4.4 모바일 레이아웃 그리드

| 항목 | 값 |
|---|---|
| 좌우 안전 패딩 (모바일) | 16px (`space-4`) |
| 콘텐츠 최대 너비 | 단일 컬럼 (화면 너비 - 32px) |
| AppBar 높이 | 56px (Material 3 기본) |
| Bottom Navigation 높이 | **80px** (safe area 포함, 화면명세서 라인 582 검증) |
| 모달(Bottom Sheet) 최대 높이 | 화면 높이의 90% 또는 884px 중 작은 값 |

### 4.5 터치 타깃 (Touch Targets)

## 8. 상태 & 모션 (States & Motion)

### 8.1 화면별 상태 매트릭스

| 화면 | Default | Loading | Empty | Error | Offline | Success |
|---|---|---|---|---|---|---|
| 1 로그인 | 입력 폼 | 버튼 스피너 | — | 토스트 ("관리자에게 문의" 등) | (해당 없음, 인증 필수) | 화면 2로 이동 |
| 2 탱크/위치 | 카드/드롭다운 | — | 드롭다운 옵션 없음 → disable | warning "선택한 탱크 유형의 AI 모델이 준비되지 않았습니다" | — | 화면 3로 이동 |
| 3 대시보드 | 통계 + 이력 | 스켈레톤 | "최근 세션 이력 없음" | "서버에 연결할 수 없습니다..." | 오프라인 모드 배너 | 데이터 표시 |
| 4 이력 | 카드 리스트 | 로딩 인디케이터 | "오늘의 검사 결과가 없습니다" | 동상 | 동상 + 큐 카운트 | "세션 완료" 활성화 |
| 5 카메라 | 프리뷰 | 모델 로드 / 추론 중(백그라운드) | — | 권한 거부 | 로컬 큐 적재 표시 | 결과 카드 자동 추가 |
| 6 결과 처리 | 읽기 전용 | 저장 중 스피너 | — | 다이얼로그 "서버 오류가 발생했습니다..." | — | "확정 저장됨" 뱃지 + 화면 4로 이동 |
| 7 설정 | 토글/드롭다운 | 설정 로딩 | — | 토큰 만료 → 로그인 화면 | — | "설정이 저장되었습니다" 토스트 |

### 8.2 모션 토큰

| 토큰 | duration | easing | 용도 |
|---|---|---|---|
| `motion-fast` | 100ms | `cubic-bezier(0.2, 0, 0, 1)` | 아이콘 회전, 작은 토글 전환 |
| `motion-base` | 150ms | 동상 | 버튼 누름(`scale 0.98`), 탭 활성 전환 |
| `motion-slow` | 250ms | 동상 | 모달 슬라이드 업, 페이드 |
| `motion-emphasized` | 400ms | `cubic-bezier(0.05, 0.7, 0.1, 1)` | 첫 등장 애니메이션, 카드 진입 |

### 8.3 인터랙션 패턴
- **버튼 누름**: `scale(0.98)`, `motion-base` (150ms)
- **탭 활성 전환**: 배경/텍스트 색상 + `radius-lg`, `motion-base`
- **Bottom Sheet 등장**: Flutter `showModalBottomSheet` 기본 (slide-up), `motion-slow`
- **카메라 셔터 피드백**: 짧은 화이트 플래시 오버레이 100ms
- **ProgressBar 채움**: 0 → target%, `motion-emphasized` (400ms ease-out)
- **카드 진입**: 리스트에 카드 추가될 때 fade + slide-in, `motion-base`

## 9. 접근성 (Accessibility)

### 9.1 컬러 & 대비
- 모든 토큰 페어가 **WCAG 2.1 AA (4.5:1)** 기준 충족 (§2.7 검증표)
- 양품/불량 판정은 색상 단독이 아닌 **색상 + 아이콘 + 라벨** 의 3중 표기 (원칙 P2)

### 9.2 터치 타깃
- 최소 48 × 48 (M3 가이드라인)
- **권장 56 × 56 (장갑 착용 현장 환경)**
- 인접 터치 타깃 간 최소 8px 간격 (`space-2`)

### 9.3 현장 환경 가독성
- 본문 최소 크기 14px (`body-md`)
- 햇빛 환경 대비: surface 배경(#FFFBF5) 과 텍스트(#1F1810) 대비 15.6:1
- Grad-CAM, Top-3 등 데이터는 `code-data` 폰트로 흔들림 방지

### 9.4 스크린리더 (TalkBack)
- 모든 아이콘 버튼에 `semanticsLabel` 한국어로 제공
  - 예: `IconButton(icon: Icon(Icons.edit_location_alt), tooltip: '위치 변경')` → Flutter 가 자동 라벨링
- 상태 뱃지는 텍스트로도 의미 전달 ("검사 완료" 등)
- 색상바는 시각적 강조이며, 카드 본문에 판정 텍스트가 함께 노출됨

### 9.5 키보드/포커스 (참고)
- 모바일 우선 시스템이므로 키보드 네비게이션은 입력 폼 내부에서만 고려
- TextField 사이 Tab 전환은 Flutter 기본 동작 활용

### 9.6 다국어 (Post-MVP)
- 설정에서 ko / en 토글
- Pretendard 단독 사용 시 영문 글리프도 동일 폰트로 처리됨 (페어링 불필요)
- 1주차 MVP 는 한국어만 적용

## 11. Post-MVP 및 미정 사항

| 항목 | 사유 / 처리 |
|---|---|
| **다크 모드 토큰** | 1주차 MVP 범위 외. M3 표준 다크 팔레트 가이드라인을 따라 추후 정의 |
| **다국어 (영문 UI) 폰트 처리** | Pretendard 단독 사용 시 글리프 자체 지원. 영문 UI 전용 디자인 검토는 Post-MVP |
| **정식 모션 시스템** | M3 motion easing curves 풀세트 (linear, standard, emphasized-decelerate 등) 적용은 Post-MVP |
| **일러스트레이션 가이드** | EmptyState 그래픽, 온보딩 일러스트 등 — Post-MVP |
| **정식 브랜드 로고** | 현재 Material Symbols `factory` 아이콘 사용. 정식 로고 디자인 도입은 Post-MVP |
| **푸시 알림 디자인** | UI 토글만 1주차 적용, 알림 카드·인앱 알림 디자인은 Post-MVP |
| **태블릿/랜드스케이프 레이아웃** | 현재 모바일 포트레이트 전용. 태블릿 대응은 Post-MVP |
